# Maintainer Response SLA

This document defines the expected response times for different types of issues and interactions in the Desygn AI repository. These are targets, not guarantees.

## Response Times

| Type | Target | Max |
|------|--------|-----|
| Security issues | 24 hours | 48 hours |
| Bug reports (critical) | 48 hours | 1 week |
| Bug reports (other) | 1 week | 2 weeks |
| Feature requests | 2 weeks | 1 month |
| Pull requests | 3 business days | 1 week |
| Questions / Discussions | 1 week | 2 weeks |

## Expectations

- **Response does not equal resolution.** A response means a maintainer has acknowledged the issue and provided initial feedback or triage — not that the issue has been fixed or implemented.
- **Maintainers are volunteers.** This is an open source project maintained in personal time. Response times may vary during holidays, personal leave, or high-activity periods.
- **Priority labels guide urgency.** Issues without a priority label will be triaged in the order received. Adding clear reproduction steps and context helps maintainers prioritize faster.
- **Security issues follow a separate process.** Refer to [SECURITY.md](../SECURITY.md) for responsible disclosure instructions. Do not open public issues for security vulnerabilities.

## Labels

The following labels are used to communicate status and priority across issues and pull requests:

| Label | Description |
|-------|-------------|
| `good-first-issue` | Beginner-friendly — well-scoped with clear guidance |
| `help-wanted` | Community help welcome — maintainers may not have bandwidth |
| `priority:critical` | Needs immediate attention — blocking or data-loss risk |
| `priority:high` | Important — fix or review soon |
| `priority:medium` | Normal priority — will be addressed in upcoming cycles |
| `priority:low` | Nice to have — addressed when bandwidth allows |
| `wontfix` | Out of scope or intentionally not planned |
| `duplicate` | Already reported — see linked issue |

## Escalation

If a response has not been received within the maximum time listed above, you may:

1. Add a comment on the issue to gently ping the maintainers.
2. Open a Discussion in the repository to bring broader attention.
3. Reach out via the project's community channels if available.

We appreciate your patience and contributions to keeping Desygn AI healthy.
